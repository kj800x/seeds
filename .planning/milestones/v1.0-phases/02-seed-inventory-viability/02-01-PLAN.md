  <action>
1. Create `seeds-rs/src/viability/` directory with `mod.rs` and `lookup.rs`.

2. In `lookup.rs`, create the species lookup table using `std::sync::LazyLock<HashMap<&'static str, u8>>` with `VIABILITY_TABLE` containing the full dataset from the research document (~80 entries covering vegetables, herbs, flowers). All keys lowercase. Set `DEFAULT_MAX_YEARS: u8 = 2`.

   Implement `lookup_max_years(subcategory: Option<&str>, category: Option<&str>) -> u8`:
   - First try subcategory: normalize to lowercase, strip "SubCat - " or "subcat - " prefix if present (Botanical Interests tags format), then look up in table.
   - If no match, try category the same way.
   - If still no match, return DEFAULT_MAX_YEARS.

3. In `mod.rs`, define:
   ```rust
   pub struct ViabilityEstimate {
       pub percentage: u8,
       pub max_years: u8,
       pub age_years: u8,
       pub species_key: String,
   }
   ```

   Implement `estimate_viability(subcategory: Option<&str>, category: Option<&str>, purchase_year: Option<i64>) -> Option<ViabilityEstimate>`:
   - Return None if purchase_year is None.
   - Get current year from `chrono::Local::now().year()`.
   - Compute age = (current_year - purchase_year).max(0) as u8.
   - Call `lookup_max_years` for max_years.
   - If subcategory and category are both None, return None (can't determine species).
   - If age == 0, percentage = 100.
   - If age >= max_years, percentage = 0.
   - Otherwise linear: percentage = ((max_years - age) as f32 / max_years as f32 * 100.0) as u8.
   - Return Some(ViabilityEstimate { ... }).

4. Register the viability module in `main.rs`: Add `mod viability;` to the module declarations.

5. Write tests in `viability/mod.rs` using `#[cfg(test)] mod tests`. Test the key behaviors listed above.
