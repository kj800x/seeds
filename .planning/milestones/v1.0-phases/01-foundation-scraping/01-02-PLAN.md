2. Create seeds-rs/src/scraper/parser.rs:
   - pub fn parse_tags(tags_str: &str) -> ParsedTags struct
   - ParsedTags: category, subcategory, light_requirement, frost_tolerance (all Option<String>), is_organic: bool, is_heirloom: bool
   - Parse "Cat - X" -> category, "SubCat - X" -> subcategory
   - Parse light patterns like "Full Sun", "Full Sun to Part Shade" -> light_requirement
   - Parse "Frost Tolerant" -> frost_tolerance
   - Parse "organic" / "heirloom-*" tags for booleans
   - pub fn parse_growing_details(html: &str) -> GrowingDetails struct
   - GrowingDetails: days_to_maturity, sow_depth, plant_spacing, germination_info, planting_instructions, growing_instructions, harvest_instructions (all Option<String>)
   - **CRITICAL: The exact HTML selectors for growing details are unknown (LOW confidence from research).** The implementation approach:
     a. First, fetch a real BI product page with reqwest and inspect the raw HTML
     b. Look for common patterns: sections with class names like "product-details", "growing-info", tabbed content areas, or metafield-rendered sections
     c. Use scraper crate with CSS selectors to extract text content from identified sections
     d. If growing details can't be reliably extracted on first attempt, return None for all fields -- the raw HTML is stored (SCRP-05) so parsing can be refined later
     e. Log warnings via tracing when expected sections aren't found
   - All fields must be Option<String> to handle missing data gracefully (Pitfall 7)

3. Create seeds-rs/src/scraper/images.rs:
   - pub async fn download_images(client: &reqwest::Client, images: &[ShopifyImage], seed_id: i64, data_dir: &Path) -> Vec<DownloadedImage>
   - DownloadedImage struct: position (i32), filename (String), width/height (Option<i32>)
   - Create data_dir/images/{seed_id}/ directory
   - Download each image individually with error handling -- partial success is OK (Pitfall 4)
   - Detect content-type from response headers for file extension (default to .jpg)
   - Image at position 1 = hero/main image (per RESEARCH.md "Main Image" heuristic)
   - Log warnings for failed downloads but continue

4. Create seeds-rs/src/scraper/mod.rs:
   - Re-export fetcher, parser, images
   - pub async fn scrape_and_save(state: &AppState, url: &str) -> Result<i64, AppError> -- the main orchestrator function:
     a. Build reqwest::Client
     b. Extract handle from URL, check for duplicate via find_seed_by_handle
     c. Call fetch_product to get JSON + HTML
     d. Call parse_tags on the tags string
     e. Call parse_growing_details on the HTML
     f. Call insert_seed with all data (including raw_html)
     g. Call download_images
     h. Call insert_image for each downloaded image
     i. Return the new seed id
   - Define a ScrapeError variant for duplicate URLs that carries the existing seed_id
